# Complaint Escalation Standard

Article ID: KB-0009
Category: complaint escalation
Audience: Frontline Agent
Last reviewed: 2026-05-22

## Purpose
Customer expresses dissatisfaction, regulatory concern, hardship, or repeated unresolved contact.

## Agent Procedure
1. Confirm the customer's identity using approved synthetic verification steps.
2. Review account status, case history, and related transactions.
3. Apply the category policy: Acknowledge complaint, avoid defensiveness, create escalation record, and set follow-up window.
4. Document the decision, evidence, and any handoff queue.

## Customer-Safe Language
Use plain language. Acknowledge the concern. Explain the next step without promising outcomes that require operations review.

## Escalation
Escalate to Executive Resolution when documentation is incomplete, customer sentiment is high-risk, or the request exceeds frontline authority.
